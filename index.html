Project


Starbucks Locator

In this project, I built an interactive web mapping application using open source data. I manipulated, organized and stored open source data in PostgreSQL Database, then published geospatial data through GeoServer. This application has a query bar, which is used to search service, products, or spatial location. Some functionalities, implemented in this application like changing base map layer, visible or invisible Starbucks display, query information about Starbucks. This application is coded with JavaScript, Leaflet, jQuery, CSS, Bootstrap, Mapbox and Google Maps API.


Mini GIS application

This project was written using Python to find intersecting points and segments of United States Highways and major rivers. Some functionalities like Zoom In, Zoom Out, Pan, Check Intersection, Check Intersecting Segments and Quit were implemented. I used Tkinter to visualize the user interface.


Social Media Analysis

This project examined the number of tweets changing through the festival period, the relationship between Twitter involved and socioeconomic variables. Twitter data was collected in two weeks around 2014 Spring Festival in New York City. I used some key words like "Spring Festival", "Chinese new year" to filter data, then imported these tweets into CartoDB. A heat map varying through timeline was created. Additionally, I created a tagged cloud in Gephi to identify keywords people concern about during Spring Festival.
